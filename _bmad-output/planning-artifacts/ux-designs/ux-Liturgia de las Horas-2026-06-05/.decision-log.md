# UX Decision Log — Liturgia de las Horas

## Discovery Summary (2026-06-05)

**Visual Feel:** Warm and traditional  
**Color Palette:** [ASSUMPTION] Warm neutrals (cream, soft brown, warm gray) with liturgical season accents  
**Form Factor:** Mobile-first (desktop responsive)  
**Accessibility:** Keyboard navigation + screen reader (from architecture)  
**Inspiration:** None specific — will derive from liturgical reverence and prayer-card tradition

---

## Key Decisions

1. **Mobile-first layout** — All primary flows (hour selection, prayer view) optimized for small screens; desktop gets breathing room via wider cards and sidebars.

2. **Warm color palette** — Cream backgrounds (#f5f1e8), warm gray text (#4a4a46), soft browns and golds for liturgical accents. Respect `data-season` CSS vars from architecture (purple for Advent, white for Christmas, green for Ordinary Time, etc.).

3. **Traditional typography** — Serif for prayer text (reverent, contemplative), clean sans-serif for UI labels and buttons. Generous line-height (1.8) for readability and breathing room.

4. **Card-based UI** — Prayer cards are the core interaction; each section (Invitatorio, Himno, Salmodia, etc.) is a distinct, scrollable card. Minimalist aesthetic: no heavy shadows, subtle spacing and borders.

5. **Prayer-book aesthetic** — Design echoes physical liturgy books: clean margins, contemplative white space, focus on text without visual noise.

---

## Form Factor Closure

**Mobile (primary):** Stacked cards, full-width buttons, large touch targets (44px+), single-column layout.  
**Desktop (secondary):** Optional sidebar for date/hour nav, cards in a narrower well-proportioned column, side-by-side controls.

**No form factor ambiguity:** Mobile is primary; desktop is auto-layout enhancement. Confirmed.

## Open Items & Assumptions

### Phase-Blockers (address before implementation)

None. Spines are complete and implementation-ready.

### Non-Blockers (log for post-MVP)

1. **Info icon tooltip style** — Small dark overlay with white text, positioned dynamically. Style in DESIGN.md before component build.
2. **Nearest hour highlight** — Uses `soft_brown` background + white text. Verify UX in browser (may add subtle glow/animation post-MVP).
3. **Date picker UI** — Inline native `<input type="date">` for MVP; redesign desktop variant post-MVP.
4. **Swipe gestures** — Advanced mobile interaction (advance card, return to home). Deferred; keyboard/tap primary.
5. **Calendar view entry point** — Tab or menu to reach `/#/calendar`. Minor nav detail; implement per architecture.
6. **Auto-advance audio** — When card finishes, auto-show next card. Nice-to-have; test with users first.
7. **Progress indicators** — Card count ("Himno, 2 of 5") shown in EXPERIENCE.md. Confirm placement in HourView layout.
8. **Error states** — Unlikely (romcal always has data). Fallback copy: "Disculpa, hubo un problema. Por favor, recarga la página."
9. **Loading states** — Skeleton placeholders for LiturgicalHeader while romcal computes. Optional polish.
10. **Keyboard shortcuts** — Arrow keys for card nav (post-MVP). Tab + Enter primary for MVP.

### Resolved Assumptions

- **Form factor:** Mobile-first, responsive desktop. Closed.
- **Color palette:** Warm neutrals + season accents. Closed.
- **Typography:** Serif for prayer, sans for UI. Closed.
- **Accessibility:** Keyboard nav + screen reader (from architecture). Closed.
- **Dark mode:** Respects `prefers-color-scheme`, no toggle MVP. Closed.
